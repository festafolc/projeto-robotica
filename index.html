UR3, 6DOF, standard_DH (mdl_ur3)


O robô ur3 é considerado antropomórfico, isto é, tem características parecidas aos humanos. Este robô com 6 juntas pode funcionar como um braço humano com a característica especial do movimento circular do punho.

Inicializamos o robot ur3.

>> mdl_ur3


Vemos as características do robô.

>> ur3



A tabela oferece uma descrição detalhada do robô segundo os parâmetros Denavit-Hartenberg. Este tipo de notação é um método para conhecer a relação da cinemática entre dois elos conectados. Assim, este método é baseado na representação da posição e orientação de um corpo rígido.

Pode-se ver que tem 6 juntas de rotação indicadas pela letra ‘R’. É possível ver o número específico de juntas com o comando de abaixo.

>> ur3.n



Por otra parte, podemos ver as juntas representadas em tipo de objetos vector que conformam o robô.

>> links = ur3.links



Para visualizar o robô fazemos plot() e um vector de posição ‘q’, por exemplo qz = [0 0 0 0 0 0] que já veio no momento de inicialização do robô.

>> ur3.plot(qz)



Para calcular a cinemática directa (forward kinematic) utlizamos o método fkine() que devolve a matriz de transformação homogênea correspondente à posição do end-effector. Podemos observar esta transformação com algum vector dado, por exemplo.

>> ur3.fkine(qz)



>> ur3.fkine(qr)



Onde ‘qr’ tem os seguintes valores.

>> qr



Observemos agora o robô com os valores de ‘qr’

>> ur3.plot(qr)



Com o método teach() podemos também visualizar o robô e alterar os valores das juntas gráficamente. De forma predefinida o vector de posiçao que utliza é [0 0 0 0 0 0].

>> ur3.teach()



Com este gráfico podemos observar que o end-effector (q6) rota em torno ao eixo z completando uma volta completa.

Existem outras formas de mostrar as interfaces. Assim, no GitHub do PeterCorke (https://github.com/petercorke/robotics-toolbox-matlab/blob/master/%40SerialLink/SerialLink.m) mostra os seguintes displays para interfaces. Também no página do Peter Cork sobre a documentação do SerialLink (https://www.petercorke.com/RTB/r9/html/SerialLink.html) podemos ver que há opções para o método teach().


Podemos ver um exemplo.

>> ur3.teach(‘rpy’)



Para calcular a cinemática inversa é preciso conhecer a cinemática direta. Já falamos que o método fkine() devolve a matriz de transformação homogênea dado um vector q. Esta pose é representada pela letra T. Segue um exemplo com qz.

>> mdl_ur3
>> T = ur3.fkine(qz)



Agora com o método ikine() (inverse kinematics) obtemos as coordenadas das juntas dada a matriz de transformação homogênea. No nosso caso, esta matriz foi definida como T.

>> ur3.ikine(T)



Podemos comprovar que a resposta são os valores das coordenadas do vector qz, o qual foi utilizado para encontrar a matriz de transformação homogêna.

É importante saber que também existe o método ikine6s() mas este é utilizado para robôs de punho esférico, o qual é muito comum nos robôs industriais. Também existe o método ikine3() que serve para calcular a cinemática inversa de um robô de três graus de liberdade sem punho esférico.

A seguir mostramos um exemplo de movimento que faz o robô de um vector a outro com parâmetros predefinidos previamente.
Primeiro escolhemos os vectores. No nosso caso sao os vetores qz e qr. A seguir definimos o número de vectores, isto são passos, que vai fazer o robô desde o ponto inicial até o ponto final. Quanto maior seja o número mais passos vai fazer na trajetória.

>> qt = jtraj(qz, qr, 100)
>> ur3.plot(qt)

qn = [3.1416 0 -1.5708 3.1416 0 0]; qz = [0 0 0 0 0 0]


ABB, IRB140, 6DOF, standard_DH (mdl_irb140)


O robô irb140 é considerado antropomórfico, isto é, tem características parecidas aos humanos. Este robô com 6 juntas pode funcionar como um braço humano com a característica especial do movimento circular do pulso.

Inicializamos o robô irb140 e vemos suas características.

>> mdl_irb140


Observamos que este robot tem seis graus de liberdade todos de tipo de rotação. A seguir mostramos o robô.

>> irb140.plot(qz);


Agora vamos mostrar os valores das posiçoes angulares de uma trajetória.

>> t = [0:.05:5];
>> q = jtraj(qz, qr, t)
>> figure(1);
>> subplot(2,1,1)
>> plot(t,q(:,1));
>> title('Theta');
>> xlabel('Tempo (s)');
>> ylabel('Junta 1 rad');
>> subplot(2,1,2)
>> plot(t,q(:,2));
>> title('Theta');
>> xlabel('Tempo (s)');
>> ylabel('Junta 2 rad');



>> figure(1);
>> subplot(2,1,1)
>> plot(t,q(:,3));
>> title('Theta');
>> xlabel('Tempo (s)');
>> ylabel('Junta 3 rad');
>> subplot(2,1,2)
>> plot(t,q(:,4));
>> title('Theta');
>> xlabel('Tempo (s)');
>> ylabel('Junta 4 rad');



>> figure(1);
>> subplot(2,1,1)
>> plot(t,q(:,5));
>> title('Theta');
>> xlabel('Tempo (s)');
>> ylabel('Junta 5 rad');
>> subplot(2,1,2)
>> plot(t,q(:,6));
>> title('Theta');
>> xlabel('Tempo (s)');
>> ylabel('Junta 6 rad');



Observamos que as juntas 2 e 6 pasam de 0 a –1.5 durante os 5 segundos da trajetória. Da mesma forma as juntas 3 e 5 passam de 0 a 1.5 nos valores theta.

Vamos agora ver os gráficos da velocidade e acelaração angular. Primeiro obtemos os valores da posição angular, a velocidade angular e acelaração angular.

>> [q, dq, ddq] = jtraj(qz, qr, t);
